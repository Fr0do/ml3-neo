# Judge prompt template

Этот файл — template для судейского промпта. Сейчас он не используется
напрямую: текущий `judge.py` собирает промпт инлайн в `_build_prompt`.
Файл сохранён как точка расширения, если захочется вынести промпт из кода
и иметь несколько вариантов (например, "строгий" и "мягкий" режимы) для
A/B-сравнения судей.

Структура промпта, которую формирует `judge.py`:

1. System header (роль судьи, ограничения).
2. Метаданные ДЗ (id, title, type, weight).
3. RUBRIC (yaml целиком — это критически важно: judge видит ровно то,
   что видит студент).
4. MODEL.md студента.
5. notebook.py студента (truncated до 60K символов).
6. Output instruction (формат JSON из rubric.output_format).

Если будете править промпт — правьте `_build_prompt` в `judge.py`, а
этот файл держите как живую документацию.
